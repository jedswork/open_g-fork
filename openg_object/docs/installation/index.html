OpenG Class Templates for LabVOOP

Overview

The OpenG Class Templates for LabVOOP (LabVIEW Object Oriented Programming) are a set of templates for creating classes beyond the simple by value objects provided natively in LabVIEW.

There are currently three templates:
  • queue based by-reference class
  • pure abstract by-reference class
  • active asynchronous by-reference class
Each of these class templates depends on (inherits from) an OpenG base class.  This allows the base classes to be improved without the need for users of the class templates to make the improves themselves.
There are currently two base classes:
  • OpenG object
  • OpenG reference object

Installation Instructions

1) Download and install VI Package Manager.  See these two useful links:
  • How do I use VIPM to get OpenG libraries installed onto LabVIEW?
  • Getting Started with VIPM and Installing OpenG Libraries into LabVIEW
2) Download the OpenG class templates and support library. (Note: They are not yet available for download using VIPM, but will be, after officially released.)
  • oglib_openg_object-1.0beta1-1.ogp (OpenG Base Classes for LabVIEW Object Oriented Programming)
  • ogrsc_active_obj_tmpl-1.0beta1-1.ogp (Template for active asynchronous by-reference object classes in LabVIEW Object-Oriented Programming)
  • ogrsc_pure_abstract_ref_class_tmpl-1.0beta1-1.ogp (Template for pure abstract by-reference classes)
  • ogrsc_ref_obj_tmpl-1.0beta1-1.ogp (Template for queue based by-reference classes)
3) Add OpenG object packages to your VIPM library
Open VIPM and select Package>>Add Package(s) to Package List.  Browse to the folder containing the packages that you just downloaded, select the packages, and press the "Add" button.  The packages will appear in VIPM's

4) Install the OpenG object packages
From the VIPM Package list, find the OpenG object packages.  Right-click on them and select Install.

5) Restart LabVIEW
If LabVIEW is currently running, restart LabVIEW.

Using OpenG class templates from within Endevo GDS

The OpenG class templates may be used in conjuction with the Endevo GDS tool.  This tools allows you to create new classes from the templates, edit your classes, and even create your own class templates!  We are very thankful to Endevo for thier support of OpenG and for making the GDS Community Edition version of thier tool freely available.

Obtaining the Endevo GDS Tool

The Endevo GDS Tool must be obtained from Endevo.  Visit them on the web at http://www.endevo.se for a free download of Endevo GDS Community Edition.

Configuring Endevo GDS to use the OpenG class templates

First, make sure that you have installed the Endevo GDS tool according to its installation instructions and that you have restarted LabVIEW after installing both the Endevo GDS and the OpenG class templates and libraries.

In order for the Endevo GDS to use the OpenG class templates, you will need to configure it to use the OpenG class provider.  To do this, open the GOOP Options Dialog (shown below) by selecting Tools>>GOOP>>Class Provider Options from them menu. (You can also open this dialog from within the LabVIEW Project Explorer window by pressing the button on the toolbar).



Ensure that the OpenG class provider is selected and press OK.  The Endevo GDS tool is now configured to use the OpenG class templates.
